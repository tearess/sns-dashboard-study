# SNS 대시보드 사용 설명서

이 설명서는 처음 보는 사람도 천천히 따라 할 수 있게 쓴 사용 방법입니다.  
이 프로그램은 여러 SNS에 올릴 글을 한곳에서 만들고, 저장하고, 필요하면 X(Twitter)에 바로 올릴 수 있게 도와주는 도구입니다.

## 1. 이 프로그램으로 할 수 있는 일

이 프로그램은 SNS 글쓰기 공책처럼 생각하면 됩니다.

할 수 있는 일은 다음과 같습니다.

- SNS에 올릴 글을 만든다.
- Instagram, Facebook, X(Twitter), Threads, YouTube용 글을 따로 적는다.
- 글을 초안으로 저장한다.
- 나중에 올릴 글을 예약 상태로 관리한다.
- 이미 올린 글처럼 발행완료 상태로 표시한다.
- X(Twitter)에 연결된 계정으로 글을 바로 올린다.
- OpenAI API 키가 있으면 AI로 SNS별 초안을 만들 수 있다.
- 캘린더에서 어떤 날에 어떤 글이 있는지 볼 수 있다.

## 2. 프로그램 켜는 방법

터미널에서 프로젝트 폴더로 들어간 뒤 아래 명령어를 실행합니다.

```bash
npm run dev -- --host 127.0.0.1 --port 5174
```

그 다음 브라우저에서 아래 주소를 엽니다.

```text
http://127.0.0.1:5174/
```

현재는 혼자 쓰는 설정이라 로그인 화면 없이 바로 대시보드가 열립니다.

만약 로그인 화면이 보인다면 `.env` 파일에 아래 값이 있는지 확인합니다.

```env
VITE_DISABLE_AUTH=true
```

## 3. 화면 구성

왼쪽에는 메뉴가 있습니다.

- 홈: 전체 성과와 요약을 보는 곳
- 콘텐츠 관리: SNS 글을 만들고 관리하는 곳
- 커뮤니티: 댓글이나 반응을 보는 곳
- 리서치: 아이디어와 경쟁사 정보를 보는 곳
- 연동 관리: X, OpenAI 같은 외부 서비스 키를 저장하는 곳
- 회원 관리: 여러 사람이 함께 쓸 때 회원을 관리하는 곳

지금 가장 많이 쓰는 메뉴는 `콘텐츠 관리`와 `연동 관리`입니다.

## 4. SNS 글 새로 만들기

1. 왼쪽 메뉴에서 `콘텐츠 관리`를 누릅니다.
2. 오른쪽 위의 `새 콘텐츠` 버튼을 누릅니다.
3. 제목을 입력합니다.
4. 마스터 글을 입력합니다.

마스터 글은 모든 SNS 글의 원본이 되는 글입니다.  
예를 들어 "AI 시대 X 마케팅 전략"에 대한 긴 설명을 먼저 적어두고, 그 내용을 X용, Instagram용, Facebook용으로 바꿔 쓰면 됩니다.

## 5. 플랫폼 고르기

콘텐츠를 만들 때 어떤 SNS에 올릴지 고를 수 있습니다.

예를 들어 X에만 올리고 싶다면 X(Twitter)만 선택하면 됩니다.

여러 곳에 올리고 싶다면 다음처럼 여러 개를 고를 수 있습니다.

- X(Twitter)
- Instagram
- Facebook
- Threads
- YouTube

선택한 플랫폼마다 따로 글을 적을 수 있습니다.

## 6. SNS별 글 적기

콘텐츠 편집 화면에는 플랫폼별 입력 칸이 있습니다.

예를 들어 X(Twitter) 칸에는 X에 올릴 짧은 글을 적습니다.  
Instagram 칸에는 조금 더 감성적인 글이나 해시태그가 많은 글을 적을 수 있습니다.  
YouTube 칸에는 영상 제목이나 설명처럼 조금 긴 글을 적을 수 있습니다.

중요한 점:

- X는 글자 수가 너무 길면 올리기 어렵습니다.
- SNS마다 말투가 조금 달라도 괜찮습니다.
- 같은 내용을 그대로 복사하기보다 플랫폼에 맞게 조금씩 바꾸는 것이 좋습니다.

## 7. 저장하기

글을 다 적었다면 저장 버튼을 누릅니다.

저장하면 콘텐츠 목록에서 다시 볼 수 있습니다.

상태는 보통 아래 중 하나입니다.

- 초안: 아직 준비 중인 글
- 예약: 나중에 올릴 글
- 발행완료: 이미 올린 글

## 8. X(Twitter)에 바로 올리기

X에 실제로 글을 올리려면 먼저 X 연결 정보가 저장되어 있어야 합니다.

### 8-1. X 키 저장하기

1. 왼쪽 메뉴에서 `연동 관리`를 누릅니다.
2. `SNS 연동` 탭을 엽니다.
3. `X (Twitter)` 항목을 찾습니다.
4. 아래 값을 입력합니다.

필요한 값:

- Consumer Key
- Consumer Key Secret
- Access Token
- Access Token Secret

5. 저장 버튼을 누릅니다.

이 값들은 비밀번호처럼 중요한 값입니다. 다른 사람에게 보여주면 안 됩니다.

### 8-2. X에 게시하기

1. `콘텐츠 관리`로 갑니다.
2. X에 올릴 콘텐츠를 엽니다.
3. X(Twitter) 글이 적혀 있는지 확인합니다.
4. `즉시 발행` 또는 X 발행 버튼을 누릅니다.
5. 성공하면 X 게시 URL이 결과에 표시됩니다.

주의:

버튼을 누르면 실제 X 계정에 글이 올라갈 수 있습니다.  
글 내용을 꼭 다시 확인하고 누르세요.

## 9. AI로 초안 만들기

AI 초안 생성을 쓰려면 OpenAI API 키가 필요합니다.

### 9-1. OpenAI 키 저장하기

1. 왼쪽 메뉴에서 `연동 관리`를 누릅니다.
2. `서비스 연동` 탭을 엽니다.
3. `OpenAI` 항목을 찾습니다.
4. API 키를 입력하고 저장합니다.

API 키도 비밀번호처럼 중요한 값입니다.

### 9-2. AI 초안 생성하기

1. `콘텐츠 관리`에서 새 콘텐츠를 만듭니다.
2. 마스터 글을 입력합니다.
3. 어떤 플랫폼용 글을 만들지 고릅니다.
4. 프롬프트를 선택하거나 직접 씁니다.
5. 초안 생성 버튼을 누릅니다.
6. AI가 플랫폼별 글을 만들어 줍니다.
7. 결과를 읽어보고 이상한 부분은 직접 고칩니다.

AI가 만든 글은 바로 믿고 올리기보다 사람이 한 번 읽어보는 것이 좋습니다.

## 10. 콘텐츠 찾기

콘텐츠가 많아지면 검색을 사용합니다.

검색할 수 있는 것:

- 제목
- 플랫폼
- 상태
- 날짜
- 등록자

예를 들어 X 글만 보고 싶다면 플랫폼에서 X(Twitter)만 선택하고 검색하면 됩니다.

## 11. 캘린더 보기

콘텐츠 관리 화면 아래쪽에는 캘린더가 있습니다.

캘린더에서는 어느 날짜에 어떤 글이 있는지 볼 수 있습니다.

활용 방법:

- 이번 주에 올릴 글이 충분한지 확인하기
- 특정 날짜에 글이 너무 몰려 있지 않은지 확인하기
- 예약 글을 날짜별로 확인하기

## 12. 글 수정하기

1. `콘텐츠 관리`로 갑니다.
2. 콘텐츠 목록에서 수정할 글을 찾습니다.
3. `편집` 버튼을 누릅니다.
4. 제목, 마스터 글, SNS별 글을 수정합니다.
5. 다시 저장합니다.

## 13. 글 삭제하기

1. `콘텐츠 관리`에서 지울 글을 찾습니다.
2. `삭제` 버튼을 누릅니다.
3. 확인 창이 뜨면 정말 지울지 확인합니다.

삭제한 글은 다시 되돌리기 어려울 수 있으니 조심해서 누르세요.

## 14. 자주 생기는 문제

### 화면이 하얗게 보여요

아래를 확인합니다.

1. 개발 서버가 켜져 있는지 확인합니다.
2. 브라우저를 새로고침합니다.
3. 터미널에서 에러가 있는지 확인합니다.
4. `.env` 값이 바뀌었다면 개발 서버를 다시 시작합니다.

다시 시작 명령어:

```bash
npm run dev -- --host 127.0.0.1 --port 5174
```

### 로그인 화면이 보여요

현재 혼자 쓰는 설정이라 로그인 화면이 나오지 않아야 합니다.

`.env`에 아래 값이 있는지 확인합니다.

```env
VITE_DISABLE_AUTH=true
```

값을 바꿨다면 개발 서버를 다시 시작하거나 브라우저를 새로고침합니다.

### X에 글이 안 올라가요

아래를 확인합니다.

1. `연동 관리 -> SNS 연동 -> X (Twitter)`에 키가 저장되어 있는지 확인합니다.
2. X 글 칸에 글이 비어 있지 않은지 확인합니다.
3. X 개발자 계정의 API 권한이 게시를 허용하는지 확인합니다.
4. Supabase Edge Function `post-x`가 배포되어 있는지 확인합니다.

키가 없으면 이런 메시지가 나올 수 있습니다.

```text
X 인증키를 찾을 수 없습니다. 연동 관리에서 저장해주세요.
```

만약 아래와 비슷한 메시지가 나오면 Supabase에 필요한 테이블이 아직 없는 상태입니다.

```text
Could not find the table 'public.sns_credentials' in the schema cache
```

이때는 Supabase Dashboard에서 `SQL Editor`를 열고 `supabase/setup.sql` 파일 내용을 실행한 뒤 다시 저장합니다.

### AI 초안 생성이 안 돼요

아래를 확인합니다.

1. OpenAI API 키가 저장되어 있는지 확인합니다.
2. 마스터 글이 비어 있지 않은지 확인합니다.
3. 프롬프트가 선택되어 있거나 입력되어 있는지 확인합니다.
4. 인터넷 연결을 확인합니다.

## 15. 중요한 주의사항

이 프로그램은 실제 SNS 계정과 연결될 수 있습니다.

그래서 다음을 꼭 지켜야 합니다.

- X 키와 OpenAI 키는 다른 사람에게 보여주지 않습니다.
- 게시 버튼을 누르기 전에 글을 꼭 다시 읽습니다.
- 공개 URL로 배포할 때는 로그인 기능을 다시 켜는 것이 좋습니다.
- 혼자 로컬에서 쓸 때만 `VITE_DISABLE_AUTH=true`를 사용합니다.

## 16. 처음부터 끝까지 예시

예시 목표:  
`AI 시대 X 마케팅 전략`이라는 글을 X에 올리기

순서:

1. `http://127.0.0.1:5174/`에 접속합니다.
2. `콘텐츠 관리`로 갑니다.
3. `새 콘텐츠`를 누릅니다.
4. 제목에 `AI 시대 X 마케팅 전략`을 입력합니다.
5. 마스터 글에 하고 싶은 이야기를 적습니다.
6. 플랫폼에서 X(Twitter)를 선택합니다.
7. X 글 칸에 게시할 문장을 적습니다.
8. 저장합니다.
9. 글을 다시 읽고 오타가 없는지 확인합니다.
10. `즉시 발행`을 누릅니다.
11. 성공 메시지나 X 게시 URL을 확인합니다.

## 17. 한 줄 요약

이 프로그램은 SNS 글을 미리 만들고, SNS별로 다듬고, 필요하면 X에 바로 올릴 수 있게 도와주는 개인용 대시보드입니다.
